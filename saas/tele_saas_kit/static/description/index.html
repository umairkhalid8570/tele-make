Deploy on Self & Remote server

Running Multiple Tele Instances in a single server slows down its performance. So, if you have multiple clients, then you can also deploy their instance in a remote server instead of the self server with the help of Tele Saas Kit All in One Module.

The module facilitates you to choose between Self and Remote server as per the requirements of your business.

Running Tele in Remote server means that you can segregate the servers for you(admin) and your clients for fast processing.

Add Subdomains(Domain Masking)

The module allows you to hide the actual domain name of a website from the URL field of a user's web browser in favour of another name. In simple words, multiple domains can be added for the client's instance.

Moreover, Admin has the flexibility to create a custom domain for the client as per the requirement.

Deploy Instance in any Tele Version

The module Tele Saas Kit All in One allows you to deploy Tele Instance of your client in any Tele Version. It means that no matter in which version your Tele instance is running, you can deploy your Client's instance in any other Tele Version.

Let's Understand this with an example:

Suppose Tele Saas Kit is installed in your Tele Version 13 and your clients want to run their Tele in any other version, say V14, V15 or V12 then you can deploy their version in any asked Tele Version using the custom plan addon of saas.

Customers can Manage the Backup

It offers the flexibility to the clients with the creation of a backup process for their instance by configuring the backup cycle. The client can configure a backup cycle to create a backup as hourly, daily, weekly, monthly.

As backup starts, the list of backups will be visible on the contract portal page. Clients can download the backuped file from the contract portal page.

    Where the saas client Instance will be hosted with this Addon?

    The instance of client can be managed on the same Tele server on which Tele SAAS Kit (Tele server) is installed or the SAAS admin can choose to deploy the instance of client on a different remote server.

    Does every client will have a separate database for their Tele?

    Each client will be having a separate and independent database irrespective of the database of other clients.

    Is addition of custom domain and subdomain possible for Saas client instance?

    Yes Saas client can add subdomain for his instance and after correct mapping of custom domain admin can also add custom domain to saas client contract.

    With this addon can a saas admin provide Tele instances in different language?

    Yes with our addon saas admin can configure the language in which he wants to provide Tele to his saas client, but the configuration has to be done manually.
